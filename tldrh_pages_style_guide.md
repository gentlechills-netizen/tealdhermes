# tldrh — Style guide

tldrh pages follow the [official tldr format](https://github.com/tldr-pages/tldr/blob/main/contributing-guides/style-guide.md)
with additional Hermes-specific metadata sections.

## Template

```markdown
# command

> Short description (imperative, one line).
> Syntax: /command [args]
> Aliases: /alias1, /alias2
> Available: Hermes TUI + Gateway

> Dashboard note...          ← only if command is in Dashboard
> Additional note...         ← only if in notes.yaml

- Do something specific:

`/command {{arg}}`

- Do something else:

`/command --option`
```

## Sections (in order)

### Title — `# command`
One line. Matches the command name.

### Description — `> text`
First sentence from [Hermes docs](https://hermes-agent.nousresearch.com/docs/reference/slash-commands).
Nothing less, nothing more. No editing.

### Syntax, Aliases, Available — `> Label: value`
- Syntax: always present if command has arguments. Format: `> Syntax: /cmd [arg]`
- Aliases: only present if command has aliases. Comma-separated.
- Available: always present. `Hermes TUI` / `Hermes Gateway` / `Hermes TUI + Gateway`

### Dashboard — `> text` (optional)
Only for commands with a Dashboard menu. Redirects user to the web UI.
Generated automatically from notes.yaml.

### Notes — `> text` (optional)
Additional info from notes.yaml — cross-references, links, configuration.

## Examples

### Format
Each example is a pair: description line + command line, separated by a blank line.

```markdown
- Do something specific:

`/command {{arg}}`

- Next example:

`/command --option {{value}}`
```

### Rules
- **Imperative mood.** "Clear the conversation:" not "Clears the conversation" / "Clearing the conversation".
- **At most 5 examples.**
- **Use backticks** for the command.
- **No bold, no italics** in descriptions.
- **No sudo** — Hermes commands don't need it.

### Placeholders
Use `{{placeholder}}` syntax for values the user must provide.

| Pattern | Example |
|---------|---------|
| Single value | `{{name}}`, `{{number}}` |
| Path/file | `{{path/to/directory}}` |
| Mutually exclusive | `{{on/off}}`, `{{here/global}}` |
| Descriptive | `{{what to learn}}`, `{{authentication topic}}` |

Only use placeholders when the user MUST replace the value.
If the example works as-is (e.g. `/compress --preview`), keep it literal.

## Page header (generated automatically)
The `> Syntax`, `> Aliases`, `> Available`, `> Dashboard`, and `> Notes` lines
are generated by `generate_pages.py` from COMMAND_REGISTRY, notes.yaml,
and config.yaml. **Do not edit them manually** — they are overwritten on
every `update.sh` run.

## Editing examples
Examples live in `examples.yaml`. Each entry:

```yaml
command-name:
  - desc: "Do something:"
    cmd: "/command {{arg}}"
  - desc: "Do something else:"
    cmd: "/command --option"
```

Add, edit, or remove entries here, then run `./update.sh`.
